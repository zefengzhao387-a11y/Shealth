# 2D AI智能体升级3D交互式数字人技术路线与开发实施方案

本项目旨在将现有的优雅简约 2D AI 教练界面，全面升级为具备“全自主交互、实时换装、动态性格及性别切换”能力的 3D 智能体（Digital Human Agent）。整个系统核心在于打通 **3D 网页渲染引擎**、**大语言模型（LLM）** 与 **语音合成及口型驱动（TTS & Audio2Face）** 的闭环生态。

---

## 一、 核心技术架构分析

要构建一个轻量级、低延迟且可深度定制的 Web 端 3D 数字人，系统需划分为以下四个核心解耦模块：

```text
+-----------------------------------------------------------------+
|                         用户前端界面 (Web)                      |
|   +-----------------------+         +-----------------------+   |
|   |   3D渲染与模型 (躯壳)  |         |   口型与表情同步 (神经)|   |
|   |  Three.js / R3F / VRM |         |  Visemes / BlendShapes|   |
|   +-----------+-----------+         +-----------^-----------+   |
+---------------|---------------------------------|---------------+
                | 用户输入                        | 驱动数据      
                v                                 |               
+---------------|---------------------------------|---------------+
                |             后端服务            |               |
                v                                 |               |
    +-----------+-----------+         +-----------+-----------+   |
    |   大语言模型 (大脑)   |-------->|   语音合成 TTS (喉咙) |   |
    |  System Prompt / API  | 文本回复|  Azure Neural / Audio |   |
    +-----------------------+         +-----------------------+   |
+-----------------------------------------------------------------+
```

1. **“躯壳” (3D 渲染与模型模块)**
   * **职责**：负责在网页端高效渲染 3D 角色模型，承载骨骼动画，并提供材质、衣服、性别的动态替换接口。
   * **关键点**：模型需要包含完整的面部形变键（Blend Shapes / Morph Targets），以便后续驱动面部表情与口型。

2. **“大脑” (大语言模型对话模块)**
   * **职责**：接收用户的文本或语音输入，理解健康/运动语义，并基于设定的“性格角色”生成实时文本回复。
   * **关键点**：动态管理提示词（System Prompt），根据用户前端选择的教练性格或性别，实时切算后端的 Prompt 逻辑。

3. **“喉咙” (语音合成 TTS 模块)**
   * **职责**：将 LLM 流式输出（Streaming）或全文输出的文本，转化为自然、带有情感起伏的语音音频流。
   * **关键点**：必须选择能同时输出**音频文件**和**口型时间轴数据（如 Visemes）**的 TTS 引擎。

4. **“神经” (口型与表情同步驱动模块)**
   * **职责**：作为连接“声音”与“视觉”的纽带。解析语音特征或口型数据，将其映射到 3D 模型的面部肌肉系数上，实现音画同步。

## 二、 推荐技术栈与工具选型

### 1. 前端 3D 渲染层
* **模型资产方案A（真人写实）**：**Ready Player Me / Avaturn**。直接导出标准的 `.glb`/`.gltf` 文件，原生自带 52 个标准 ARKit 面部表情基元（完美匹配口型同步）。
* **模型资产方案B（二次元/动漫风）**：**VRoid Studio**（本地客户端捏脸）。导出 `.vrm` 格式模型，在 Web 端配合生态内成熟的开源库 `@pixiv/three-vrm` 使用，断网环境下研发极度友好。
* **渲染引擎**：**Three.js** 或 **React Three Fiber (R3F)**。若前端基于 React 生态开发，推荐使用 R3F，能将 3D 组件以声明式写进 React 声明周期中，极大提升数据流转效率。

### 2. 智能大脑与性格控制层
* **大模型底座**：接入通用大模型 API（如 OpenAI GPT-4、Claude 3、通义千问、智谱 GLM 等）。
* **动态属性控制**：性格、穿搭和性别的切换逻辑设计：
  * **换装与性别**：前端点击切换时，直接通过组件更新 3D 模型的纹理材质（Texture）或重载对应的模型资产。
  * **性格切换**：前端向后端传递状态码（例如 `character_id: "gentle_female"`），后端路由动态匹配相应的 **System Prompt**（如：*“你现在是一位温柔的女性健康教练，说话风格专业且极具亲和力...”*）。

### 3. 语音与口型同步层
* **TTS 服务**：**Azure Neural TTS**（微软语音）或 **ElevenLabs**。
  * **核心技术优势**：Azure TTS 能够直接在合成音频的同时，返回一串包含时间戳的 **Viseme ID（发音素列表）**。
* **驱动原理**：
  * 前端通过 Web Audio API 播放声音。
  * 在播放的同时，开启一个 `requestAnimationFrame` 渲染帧循环，根据当前播放的音频时间戳，查找对应的 Viseme 数据。
  * 提取出对应的嘴型系数值（如 `MouthOpen`, `JawOpen`），直接修改 3D 模型网格体的 `morphTargetInfluences` 属性，让嘴部动起来。

## 三、 敏捷开发实施步骤（四阶段演进）

项目建议采取 MVP（最小可行性产品）策略逐步推进，降低多技术栈并发对接的复杂度。

### 阶段一：打通“纯文本大脑”与数据流（MVP阶段）
* [ ] **任务 1**：搭建前端基础聊天 UI 界面，后端配置大模型 API。
* [ ] **任务 2**：设计并调试 System Prompt，测试在“温柔女教练”、“硬核男教练”等不同性格设定下，大模型回复的语调、专业度是否符合预期。
* [ ] **任务 3**：实现前端能够动态向后端发送性格、性别参数，验证后端 Prompt 隔离与动态替换逻辑。

### 阶段二：引入静态 3D “躯壳”与换装（视觉呈现）
* [ ] **任务 1**：通过 VRoid Studio 或现成开源渠道，准备好两组带有完整面部 Blend Shapes 的 3D 人物模型（一男一女），导出为 `.glb`/`.vrm` 格式。
* [ ] **任务 2**：在前端项目中引入 Three.js / React Three Fiber，编写基础场景（包含相机、环境光、平行光及基础舞台底座），将模型成功渲染到网页上。
* [ ] **任务 3**：编写换装逻辑代码。通过更换衣服网格体材质的贴图颜色，或者通过控制控制隐藏/显示不同的衣服网格（Mesh），实现一键换装。

### 阶段三：攻克音画同步，赋予智能体“喉咙与神经” (核心难点)
* [ ] **任务 1**：对接 Azure 微软语音合成 API。前端向后端发送文本，后端请求 Azure 并将生成的音频流（Audio Stream）以及 **Viseme（口型映射数据集）** 组装后返回给前端。
* [ ] **任务 2**：在前端编写口型驱动控制器（LipSync Controller），将返回的 Viseme ID（0-21种基础发音嘴型）映射到 3D 模型的 Morph Targets 索引上。
* [ ] **任务 3**：联调测试。发送文本后，数字人在音频播放的同一毫秒开始起伏嘴巴，确保无肉眼可见的音画延迟。

### 阶段四：交互润色与体验优化（全面升级）
* [ ] **任务 1**：加入待机微动动画（Idle Animation）。利用骨骼动画（AnimationClip），让数字人在不说话时也有自然的呼吸、眨眼（Eye Blinking）和身体微小晃动，避免僵硬感。
* [ ] **任务 2**：情绪联动。大模型输出文本时，可以利用特定标签（如 `[开心] 看到你今天坚持运动...`），后端解析该标签，不仅驱动说话，同时触发数字人面部微笑或做出鼓励的手势。
* [ ] **任务 3**：首屏加载优化。3D 模型和骨骼资产体积较大，需加入骨骼动画文件的压缩（如 Draco 压缩），并设计优雅的加载进度条。
